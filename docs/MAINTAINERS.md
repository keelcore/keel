## Becoming a Maintainer

We welcome contributors who demonstrate sustained, high-quality engagement with the
project. The path to maintainership:

1. **Consistent contribution** — multiple merged PRs across different areas of the
   codebase, including tests and documentation.
2. **Code review participation** — constructive reviews on other contributors' PRs.
3. **Community engagement** — helping triage issues, responding to questions.
4. **Nomination** — an existing maintainer nominates the contributor via a PR to this
   file. Existing maintainers vote; a simple majority approves.

There is no minimum time requirement, only a quality and consistency bar.

## Maintainer Responsibilities

- Review and merge (or close) pull requests in a timely manner.
- Triage new issues within 5 business days.
- Participate in release planning and cut releases following the process in
  [docs/operations.md](operations.md).
- Enforce the [Code of Conduct](CODE_OF_CONDUCT.md).
- Keep the project aligned with CNCF values and processes.

## Emeritus Maintainers

Maintainers who are no longer active may move to emeritus status. Emeritus maintainers
retain their commit history attribution and are acknowledged in release notes but do
not have active repository permissions.
